[Mediapipe 라이브러리를 통한 동작 트래커 구현 보고서]

=== 1. 성공 항목 ===
- 로컬 동영상 파일 분석 : 별도 서버 없이 브라우저 내에서 즉각적인 분석 성공.
- 구글 드라이브(URL)의 동영상 파일 공유 연동: 리다이렉트를 추적하는 서버 프록시를 통해 클라우드 영상 분석 성공.
- 640px 다운스케일링: 4K 고화질 영상도 렉 없이 실시간 트래킹 (0초대 지연 시간).
- 하드웨어 가속 토글 (CPU/GPU): 시스템 사양에 따른 맞춤형 성능 제공.

=== 2. 유튜브 영상 연동 실패 및 한계점 ===
- [유튜브 직접 연동 실패]: 유튜브의 강력한 보안 정책 때문에 서버에서 영상을 직접 긁어오는 방식이 반복적으로 차단됨. 
로컬에서는 유튜브 영상에 mediapipe 구현이 되었으나, vercel에 배포시 youtube가 차단해버림. (render, railway는 아직 시도해 보지 못함)
- [Vercel 메모리/시간 제한]: Vercel 서버리스 함수 특성상 아주 큰 영상(500MB 이상) 스트리밍 시 타임아웃 발생 가능성 있음.
- [CORS 정책 차단]: 외부 URL 영상 접근 시 브라우저 보안에 막혔으나, 자체 프록시 전용 통로를 구축하여 성공함.

=== 3. 기술적 성공 요인 ===
- 오프스크린 캔버스 파이프라인: 원본 영상은 고화질로 보여주되, AI 연산은 보이지 않는 640x480 화면에서 처리하여 연산량을 1/10로 절감함.
- Redirect-Following Proxy: 구글 드라이브의 복잡한 보안 리다이렉트 주소를 서버가 끝까지 추적하여 최종 영상 데이터만 브라우저에 배달함.
- MediaPipe Task API 활용: 기존의 무거운 방식 대신 최신 Task Vision API를 사용하여 초기 모델 로딩 속도를 2배 이상 단축함.

=== 4. 향후 발전 방향 ===
- 운동 횟수 자동 카운팅: 관절 간의 각도(Vector Math)를 계산하여 스쿼트, 푸쉬업 횟수를 자동 측정하는 로직 추가. (어떻게 구현?)
- 실시간 웹캠 모드 통합: 저장된 영상뿐만 아니라 실시간 카메라 화면에서도 동일한 분석이 가능하도록 토글 버튼 구현. (어떻게  구현?)
- 데이터 엑스포트: 분석된 33개 관절 좌표를 JSON으로 추출하여 자세 교정 AI 학습 데이터로 활용. (어떻게 구현?)
- 온라인 PT(Personal Training) 트레이너 연결 : 온라인 PT 선생님과 연결 커뮤니티
- Shy 사용자를 위한 얼굴 등 캐릭터화
- 다른 사용자와 커뮤니티에 오늘의 운동 자동 인증
- 친구와 함께 1:1 영상 연결 온라인 운동.

=== 6. [webgym] 프로젝트 개발시 유의점 ===
- 보통의 일상 user는 자신의 전체 모습을 카메라로 찍지 않고, 상체 일부분만 찍을 확률이 매우 높다. 왜냐하면 전신을 카메라에 넣기 위해서는, 카메라와의 거리가 3미터 이상 떨어져야 하고, 이러면 거리때문에 사용자는 영상화면을 볼 수가 없다.
- 운동 선수의 경우 전체의 모습을 카메라로 찍을 것 같으나, 각 운동 종목에 따라 촬영각도가 너무 많을 듯 해서 구현하기 힘듦.
